data/regions.json

Graphe de régions utilisé pour le BFS d'accessibilité.

{
  "key":              "COF_1F",
  "name":             "Cave of Flame 1F",
  "dungeon":          "CoF",       // null si hors donjon
  "is_dungeon_region": true
}

Les transitions entre régions sont définies dans src/logic/rules.js via REGION_RULES :

REGION_RULES = {
  "MENU": {
    "SOUTH_FIELD": null,   // null = toujours accessible
    "COF_ENTRANCE": (inv, s) => has("Bomb Bag")(inv) || has("Gust Jar")(inv),
    ...
  }
}

data/items.json

Table de correspondance clé AP → nom d'item utilisé dans l'inventaire logique.

{
  "key":  "PROGRESSIVE_SWORD",      // Clé interne (correspond à ITEM_IMAGES dans itemImages.js)
  "name": "Progressive Sword",      // Nom AP exact (utilisé dans buildInventory)
  "ap_id": 64001                    // ID AP numérique (optionnel)
}
Le champ name doit correspondre exactement au nom utilisé dans rules.js via la fonction has(name). Toute divergence silencieuse provoque des checks toujours rouges.

src/logic/rules.js – helpers

Toutes les règles d'accessibilité sont des fonctions (inv, settings) => boolean.

Fonctions de base

FonctionSignatureDescription
has(name, n=1)(inv) => boolRetourne true si inv[name] >= n (par défaut n=1)
always() => trueRègle toujours vraie (transitions libres)
trick(key)(inv, s) => boolRetourne s.hasTrick(key)

Logique progressive scrolls

// En mode progressif, scrollLevel retourne inv["Progressive Spin Scroll"]
// En mode individuel, retourne 99 (les checks individuels s'appliquent)
const scrollLevel = (inv, s) =>
  s.progressiveScroll ? (inv["Progressive Spin Scroll"] || 0) : 99

hasSpin      = (inv, s) => has("Spin Attack")(inv)       || scrollLevel(inv, s) >= 1
hasFastSpin  = (inv, s) => has("Fast Spin Scroll")(inv)  || scrollLevel(inv, s) >= 2
hasFastSplit = (inv, s) => has("Fast Split Scroll")(inv) || scrollLevel(inv, s) >= 3
hasGreatSpin = (inv, s) => has("Greatspin")(inv)         || scrollLevel(inv, s) >= 4
hasLongSpin  = (inv, s) => has("Long Spin")(inv)         || scrollLevel(inv, s) >= 5

Structure REGION_RULES

export const REGION_RULES = {
  "MENU": {
    "REGION_A": null,                    // null = libre
    "REGION_B": always,                  // sentinel = libre
    "REGION_C": (inv, s) => hasSword(inv, s),
    "REGION_D": (inv, s) => hasSword(inv, s) || trick("my_trick")(inv, s),
  },
  "REGION_A": {
    "REGION_E": (inv, s) => has("Bomb Bag")(inv),
  }
}

Structure LOCATION_RULES

export const LOCATION_RULES = {
  "Nom exact de la location": (inv, s) => /* condition */ true,
  // Si absent → toujours accessible (règle implicite = always)
}

src/logic/accessibility.js

Algorithme en 3 passes pour calculer l'état de chaque location.

buildInventory(stateStore)

Construit l'inventaire logique depuis les items reçus AP + items manuels.

Cas spécialComportement
PROGRESSIVE_SPIN_SCROLL→ inv["Progressive Spin Scroll"] += count
PROGRESSIVE_BOMB_BAG→ inv["Bomb Bag"] += count
PROGRESSIVE_WALLET→ inv["Big Wallet"] += count
HEART_TOTAL→ inv["Heart Container"] += count
BOTTLE (count N)→ inv["Bottle 1"]…inv["Bottle N"] = 1

computeAccessibility(inv, settings)

Retourne Map<id, status> avec 3 statuts :

StatusCouleurCondition
accessible■ vertRégion atteignable + règle vraie avec inventaire actuel + settings actuels
out_of_logic■ jauneAccessible avec inventaire actuel + tous les tricks activés (mais pas avec les settings actuels)
inaccessible■ rougeNi accessible, ni out_of_logic

Les 3 passes BFS

// Passe 1 : inventaire actuel + settings actuels → accessible
const reachable         = reachableRegions(inv, settings)

// Passe 2 : inventaire actuel + TOUS les tricks → OOL candidat
const reachableOOL      = reachableRegions(inv, allTricks)

// Passe 3 : inventaire VIDE + TOUS les tricks (calculé, réservé pour usage futur)
const reachableOOLEmpty = reachableRegions({}, allTricks)

// Résultat
if (inLogic) → 'accessible'
else if (ool) → 'out_of_logic'
else          → 'inaccessible'

src/metadata/itemImages.js

ITEM_IMAGES

Dictionnaire clé → image(s) pour les cellules de la grille.

// Item simple (toggle on/off)
LANTERN: "Lamp.png"

// Item progressif (tableau d'images, clic = niveau suivant)
PROGRESSIVE_SWORD: ["Smith's Sword.png", "Green Sword.png", "Red Sword.png", "Blue Sword.png", "Four Sword.png"]

// Item compteur (ITEM_MAX_COUNT requis, clic = +1 % max+1)
BOTTLE: "Empty Bottle.png"   // + ITEM_MAX_COUNT.BOTTLE = 4

// Compteur automatique (PROGRESSIVE_COUNT_SCROLL)
// Non cliquable, affiche le nombre de scrolls actifs
PROGRESSIVE_COUNT_SCROLL: "Tiger Scroll.png"

ITEM_MAX_COUNT

Définit le maximum pour les items compteurs. Si absent → l'item est un toggle (0/1) ou progressif (tableau).
